## 💪 Технические навыки и опыт работы

##### 1. Протоколы и архитектура

- понимание принципов клиент‑серверной архитектуры;

- знание основ HTTP/HTTPS.

##### 2. Инструменты браузера

- работа с DevTools (вкладки Network, Applications);

- анализ Cookie, Local Storage, Session Storage;

- проверка кэша браузера.

##### 3. Тестирование API

- ручное тестирование REST API микросервисов;

- работа с документацией в Swagger;

- использование Postman (коллекции, переменные, cUrl, простые автотесты на JavaScript);

- опыт работы с WSDL и протоколом SOAP.

##### 4. Тестовая документация и процессы

- тестирование требований к новым фичам;

- применение техник тест‑дизайна;

- создание тестовой документации с нуля;

- составление чек‑листов в Miro (в формате Mindmap) — https://miro.com/app/board/uXjVJ2Me-_c=/?share_link_id=909644616606;

- разработка тест‑кейсов в TMS Test IT;

- формирование тест‑сьютов для регрессионного тестирования.

##### 5. Анализ данных и метрики

- работа с Яндекс Метрикой (анализ популярных браузеров, устройств и ОС);

- проверка вёрстки и стилей через Figma.

##### 6. Отчётность и баг‑трекинг

- создание баг‑репортов в Яндекс Трекер;

- использование нейросетей для автоматизации рутинных задач.

##### 7. Работа с логами и мониторингом

- анализ логов в Kibana и Sentry;

- работа с Grafana (технические и продуктовые дашборды);

- просмотр трассировки в Jaeger (для интеграционного тестирования).

##### 8. Базы данных и SQL

- работа с реляционными (PostgreSQL) и нереляционными (MongoDB, Redis) БД;

- составление SQL‑запросов средней сложности (SELECT, JOIN, GROUP BY, агрегатные функции);

- применение SQL для тестирования бизнес‑логики (например, фичи «Покупка Аватара» и «Ачивки»).

##### 9. Инструменты и среды

- использование сниффера Charles (тестирование обработки ошибок, подмена контента, Breakpoints, Map local);

- тестирование асинхронного взаимодействия микросервисов через Kafka;

- работа с Git и создание репозитория;

- базовые команды Linux и анализ логов на сервере;

- работа с GitLab CI/CD (сборка в PROD и STAGE);

- использование переменных Environment в Postman для переключения между окружениями;

- создание Mock Server (тестирование фронтенда при отсутствии бэкенда).

##### 10. Мобильная разработка и тестирование

- запуск приложения на эмуляторе Android Studio, анализ логов в Logcat;

- работа с Firebase (анализ Crash и ANR‑ошибок, просмотр логов);

- тестирование релизных и дебажных сборок;

- проверка PUSH‑сообщений и функционала диплинок.
